# OctoAcme — Role Checklists & Templates

Purpose: Provide quick, copyable checklists per role to be attached to project one-pagers, kickoff notes, or release plans.

## UX Designer — Pre-planning checklist
- [ ] Problem / hypothesis validated with user research (or planned)
- [ ] Key user flows documented and wireframes available
- [ ] Usability acceptance criteria drafted
- [ ] Prototype or UI specs attached to backlog items

## Data Analyst — Measurement checklist
- [ ] Success metrics defined with owners
- [ ] Metric spec document created (event names, properties, calculations)
- [ ] Instrumentation tickets created and assigned to engineering
- [ ] Dashboards configured and shared with stakeholders

## Release Manager — Release readiness checklist
- [ ] Release notes drafted and reviewed
- [ ] Deployment window and rollback plan confirmed
- [ ] Smoke tests and verification steps documented
- [ ] Communications plan prepared (stakeholders, support, customers)

## Customer Support Lead — Incident & feedback checklist
- [ ] Escalation contact list up-to-date
- [ ] Triage template filled for major incidents
- [ ] Feedback loop defined for turning support tickets into backlog items
- [ ] Stakeholder communication plan for customer-impacting issues

## How to use
- Copy relevant role checklist into the Project One-pager "Proposed team / roles" or the kickoff notes.
- Assign a named owner for each checklist item to avoid ambiguity.
